Problem Statement

  • Information at UoS is scattered across slow portals and static PDFs.
  • Manual verification of student records and fee slips is time-consuming.
  • Standard AI models suffer from "hallucinations" without real-time data access.
  • Lack of a centralized, intelligent 24/7 support system for faculty and students.

Aim & Objectives

  • Aim: To build an "Agentic" system that acts on behalf of the user to fetch and verify data.
  • Objective 1: Integrate a hybrid MySQL/SQLite backend for verified data retrieval.
  • Objective 2: Build an Agentic RAG pipeline using dual-model orchestration.
  • Objective 3: Deploy a modern, responsive UI accessible from any device.
